# Workflow Update

Updates the agentic-workflows instruction set and applies changes that go beyond simple file replacement.

The update-check script handles file downloads by SHA-256 comparison, but some updates require additional work:
- New entries in `.claude/settings.json` (env vars, permissions)
- New symlinks or directory structures
- Workflow state schema migrations (new fields in `.workflow-state.json`)
- New MCP configs or skill registrations
- Changes to `flake.nix` that need `nix develop` re-entry

This skill handles the full update lifecycle.

## `/workflow-update`

Run a full update cycle.

**Steps:**

### Step 1: Run the update check script

```bash
bash .agentic-workflows/scripts/update-check.sh
```

Capture the full output. Parse it for:
- **Version change:** `v{old} → v{new}` or "Already up-to-date"
- **Updated files:** lines starting with `+` (e.g., `+ .agentic-workflows/fragments/workflow-rules.md`)
- **Failed files:** lines starting with `!`
- **Whether AGENTS.md was regenerated**

If the script reports "Already up-to-date" and nothing failed, inform the user and stop.

### Step 2: Summarize changes

For each updated file, read it and summarize what changed compared to what was there before. Group by category:

| Category | Files | Impact |
|----------|-------|--------|
| **Agents** | `agents/claude-code/*.md`, `agents/cursor/*.mdc`, `agents/opencode/*.md` | Agent behavior changes — new rules, new flows, updated processes |
| **Fragments** | `fragments/*.md` | Shared rules — affects AGENTS.md (already regenerated by script) |
| **Workflow** | `workflow/*.json` | Phase/gate/schema changes — may need state migration |
| **Templates** | `templates/**/*.md` | PRD/PR/track templates — affects new artifacts only |
| **Skills** | `skills/*/SKILL.md` | Skill behavior changes — new commands, updated steps |
| **Scripts** | `scripts/*.sh` | Update/generation scripts — already applied |
| **Config** | `flake.nix`, scaffolds | Dev environment or wireframe scaffolding |

Present a concise summary to the user:
```
Updated from v2.0.38 → v2.0.39 (5 files changed)

Agents:
  - frndos-orchestra: Added env var detection for Agent Teams
  - frndos-engineer: Replaced template vars with spawn prompt extraction

Fragments:
  - workflow-rules: Rewritten Agent Teams section (mailbox, shared task list)
  - wireframe-conventions: Wireframe = feature page, sub-pages not separate wireframes

Config:
  - flake.nix: Added new Python dependency
```

### Step 3: Detect and apply non-trivial changes

After file replacement, check if any updated files require additional work that the script couldn't do:

#### 3a. Settings changes (`.claude/settings.json`)

Check if any updated agent or fragment references new env vars or experimental features.

```bash
# Check for env var references in updated files
grep -r "CLAUDE_CODE_EXPERIMENTAL" .agentic-workflows/ --include="*.md" | grep -v ".git"
```

Compare against current `.claude/settings.json`. If there are env vars referenced in agents that aren't in settings:

1. Read current `.claude/settings.json` (create if missing)
2. Show user what needs to be added:
   ```
   New env var required: CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
   This enables Agent Teams for parallel implementation.
   ```
3. Ask user for confirmation before modifying settings.json
4. Merge new env vars into the existing `env` object (preserve existing entries)

#### 3b. Workflow state schema migration

If `workflow/state-schema.json` was updated:

1. Read the new schema
2. Read `.workflow-state.json` (if it exists)
3. Check for new required fields that are missing from existing state
4. If migration is needed:
   - Show user what fields are new and their defaults
   - Ask for confirmation
   - Add missing fields with sensible defaults (null for nullable, empty object/array for collections)
5. If no `.workflow-state.json` exists, skip — it will be created correctly when needed

#### 3c. New symlinks

If new agent directories or tool-specific paths were added to the manifest:

1. Check if expected symlinks exist (`.claude/agents`, `.cursor/agents`, `.opencode/agents`)
2. If any are missing or point to wrong targets, fix them
3. Report what was fixed

#### 3d. flake.nix changes

If `flake.nix` was updated:

1. Inform user: "flake.nix was updated. You'll need to re-enter the Nix dev shell for changes to take effect."
2. If currently in a Nix shell, suggest: "Run `exit` then `nix develop` to pick up the new dependencies."

#### 3e. New skills or skill changes

If any skill files were updated:

1. Inform user: "Skills were updated. Restart your agent session for changes to take effect."
2. List which skills changed and briefly what's different

### Step 4: Final verification

After applying all changes:

1. **Verify AGENTS.md is current:**
   ```bash
   bash .agentic-workflows/scripts/generate-agents.sh
   ```
2. **Verify symlinks:**
   ```bash
   [ -L .claude/agents ] && echo "✓ .claude/agents" || echo "✗ .claude/agents missing"
   [ -L .claude/skills ] && echo "✓ .claude/skills" || echo "✗ .claude/skills missing"
   [ -f .claude/settings.json ] && echo "✓ .claude/settings.json" || echo "✗ .claude/settings.json missing"
   [ -f AGENTS.md ] && echo "✓ AGENTS.md" || echo "✗ AGENTS.md missing"
   [ -f CLAUDE.md ] || [ -L CLAUDE.md ] && echo "✓ CLAUDE.md" || echo "✗ CLAUDE.md missing"
   ```
3. **Check version:**
   ```bash
   cat .agentic-workflows/.version
   ```
4. **Report:**
   ```
   Update complete! v2.0.38 → v2.0.39

   Applied automatically:
     - 5 files replaced by update script
     - AGENTS.md regenerated
     - Added CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 to settings.json

   Action needed:
     - Restart session to pick up skill changes
     - Re-enter nix develop for flake.nix changes
   ```

## `/workflow-update check`

Dry-run — check for updates without applying them.

**Steps:**
1. Fetch remote manifest only:
   ```bash
   curl -fsSL "https://raw.githubusercontent.com/alva-intelligence/agentic-workflows/main/manifest.json" 2>/dev/null
   ```
2. Compare remote version with local `.agentic-workflows/.version`
3. If different, list which files would be updated (SHA comparison)
4. Do NOT download or apply anything

## `/workflow-update verify`

Verify the current installation is healthy.

**Steps:**
1. Check all symlinks are correct
2. Check `.claude/settings.json` has required env vars
3. Check AGENTS.md matches what generate-agents.sh would produce
4. Check `.agentic-workflows/.version` matches manifest version
5. Report any issues found
